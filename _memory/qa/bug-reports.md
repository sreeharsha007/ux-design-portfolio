## Bug Report Format

When adding a new bug, use this format:

```
### BUG-[number]: [Short description]
**Severity:** BLOCKER / HIGH / MEDIUM / LOW
**Found by:** [Agent name or "Harsha"]
**Date found:** [Date]
**Steps to reproduce:**
1. [Step 1]
2. [Step 2]
**Expected behaviour:** [What should happen]
**Actual behaviour:** [What actually happens]
**Status:** Open / In progress / Fixed
```

---

## Severity Definitions

| Level | Meaning |
|-------|---------|
| **BLOCKER** | Prevents the site from working. Must be fixed before launch. |
| **HIGH** | Significant user-facing issue. Should be fixed in current sprint. |
| **MEDIUM** | Noticeable issue that doesn't break functionality. Fix soon. |
| **LOW** | Minor visual or edge case issue. Fix when convenient. |
